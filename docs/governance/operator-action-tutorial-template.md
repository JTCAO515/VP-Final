# 操作者人工配置教程模板

Status: active  
Owner: operator / delivery agent

本模板用于把 `operator-action-register.md` 中仍未解决的动作，转换成小白可以逐步执行的中文
教程。只有两种触发条件：操作者主动要求，或相关目标到达发布/验收门。教程不得包含任何真实
密钥、token、密码、连接串或私密截图。

## 使用规则

1. 一篇教程对应一个 register ID；标题必须包含 ID 和关联 Issue。
2. 开头明确功能当前是 `open`、`in-progress`、`verified` 还是 `retired`。
3. 菜单名、按钮名、输入位置和环境必须逐步写清；界面版本不确定时标注核对日期。
4. 使用 `<PLACEHOLDER_NAME>` 展示输入位置，真实值只进入获批的第三方控制台或 secret store。
5. 每一步写“做什么、为什么、预期看到什么”；禁止把创建配置等同于验证成功。
6. 完成后把真实但不敏感的证据摘要、日期、环境和验证人写回 register；不回写秘密。

## 可复制模板

```markdown
# [OA-XXX] 功能名称配置教程

关联 Issue：#___
当前状态：open | in-progress | verified | retired
最后核对日期：YYYY-MM-DD

## 1. 这一步会启用什么

- 用户可获得的能力：
- 当前为什么不可用：
- 本教程不会做什么：

## 2. 开始前准备

- 需要的账号/权限：
- 需要先完成的决策或合同：
- 需要准备的占位符名称（只写名称，不写值）：
- 操作环境：local | preview | production

## 3. 控制台逐步操作

1. 打开 `<官方控制台 URL>` 并登录。
   - 点击：`<一级菜单>` → `<二级菜单>` → `<按钮>`
   - 预期看到：`<页面标题或状态>`
2. 创建或选择 `<资源名称>`。
   - 在 `<字段标签>` 输入/选择：`<说明>`
   - 不要输入：`<禁止项>`
   - 预期看到：`<成功提示>`
3. 打开获批的 secret store。
   - 新建名称：`<PLACEHOLDER_NAME>`
   - 值：从第三方控制台取得后直接粘贴到 secret store，不经过聊天或仓库
   - 作用环境：`<环境>`
4. 重新部署或重启：`<准确操作>`

## 4. 验证（配置不等于成功）

1. 执行：`<命令或用户操作>`
2. 预期 HTTP/UI/日志结果：`<不敏感结果>`
3. 失败时应看到的诚实降级：`<错误状态>`
4. 检查秘密未泄漏：仓库、PR、Issue、日志、截图均不含值

## 5. 回滚

1. 在 `<控制台/secret store>` 禁用或删除 `<配置名称>`。
2. 恢复 `<前一部署/设置>`。
3. 验证产品进入 `<安全降级状态>`，不能回退到假成功。

## 6. 常见错误

- 错误：把值放进 `.env.example` 或 GitHub 文档。修复：只保留占位符名称。
- 错误：只在 Preview 配置却验证 Production。修复：逐环境核对。
- 错误：看到 200 就认为业务成功。修复：核对身份、持久化、回调或真实外部结果。
- 本动作特有错误：`<列出至少两条>`

## 7. 完成后登记

- register ID：OA-XXX
- 环境：
- 验证日期：
- 验证人：
- 非敏感证据：
- 状态更新：verified | 仍 open（说明原因）
```

## 禁止内容

- 真实 secret、token、密码、service-role key、数据库连接串。
- 含秘密的终端输出、浏览器地址栏、二维码或控制台截图。
- “应该可以”“模型说成功”或 mock 截图作为外部验证。
- 未经操作者决策的支付主体、价格、SLA、隐私或法务承诺。
